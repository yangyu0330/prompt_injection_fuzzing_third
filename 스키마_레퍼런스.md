# 스키마 레퍼런스 (수정: 2026-04-08 14:19:45 KST)

이 문서는 현재 구현된 데이터 모델의 필드 참조 문서다. source of truth는 [`src/pi_fuzzer/models.py`](src/pi_fuzzer/models.py)이며, 필수/선택 여부는 현재 Pydantic 모델 정의 기준이다.

## 1. 공통 주의사항

- `TemplateRecord`, `CaseRecord`, `RunRecord`, `TargetConfig`, `Scorecard`는 모두 `extra="allow"`다.
- 즉 여기에 없는 임의 필드가 입력에 있어도 파싱은 가능하지만, 현재 코드가 의미 있게 읽는 필드는 이 문서에 적은 범위다.
- `필수`는 모델 생성 시 값이 반드시 있어야 함을 뜻한다.
- `선택`은 default가 있는 필드다. 선택이라고 해서 의미가 없는 것은 아니며, scoring이나 validation에서 강하게 쓰이는 필드도 있다.

## 3. `TemplateRecord`

현재 `TemplateRecord`는 runtime에서 전부 소비되지는 않는다. 실제 실행 경로에서 강하게 쓰이는 필드는 `template_id`, `template_kind` 정도이고, 나머지는 authoring/provenance/reference 역할이 크다.

| 필드명 | 의미 | 필수/선택 | 현재 구현상 사용 위치 | 주의사항 |
|---|---|---|---|---|
| `template_id` | template 식별자 | 필수 | build 출력, case-template 연결, CLI layer 분기 join | case의 `template_id`와 일치해야 한다 |

## 4. `CaseRecord`

`CaseRecord`는 build, validation, dedup, scoring의 핵심 단위다.

### 식별과 pairing

| 필드명 | 의미 | 필수/선택 | 현재 구현상 사용 위치 | 주의사항 |
|---|---|---|---|---|
| `template_id` | 소속 template ID | 필수 | CLI run selection join | 존재하는 template를 가리켜야 한다 |
| `case_id` | case 식별자 | 필수 | build, run, score 전 단계 | dedup drop도 `case_id`로 기록된다 |
| `language` | 언어 라벨 | 필수 | split coverage, scoring `kr_en_gap`, by_lang | 현재 샘플은 `ko`, `en` 중심 |
| `paired_case_id` | 기본 pair 대상 | 선택 | `validate_pair_invariants`, KR/EN pair authoring | 양방향 backlink가 맞아야 한다 |

| `attack_subfamily` | 하위 공격군 | 필수 | runner scenario logic, vendor claim heuristic | replay/function_call 판단에 일부 사용 |

## 5. `RunRecord`
