# 문서 분류 가이드 (수정: 2026-04-08 19:02:00 KST)

이 문서는 저장소 문서를 어디에 두는지 빠르게 판단하기 위한 분류 기준이다. 현재 기준으로 루트에는 운영 기준 문서만 두고, 그 외 문서는 `문서/` 아래에 목적별로 분리한다.

## 1. 루트에 남기는 핵심 문서

루트에는 아래 성격의 문서만 둔다.

- 현재 구현 기준의 운영 문서
- 코드와 직접 대응되는 reference 문서
- 신규 작성자나 운영자가 바로 열어야 하는 문서

현재 루트 유지 대상:

- `README.md`
- `프롬프트_인젝션_퍼저_아키텍처.md`
- `프롬프트_인젝션_퍼저_설계_설명.md`
- `프롬프트_인젝션_데이터셋_설계.md`
- `카탈로그_작성_가이드.md`
- `스키마_레퍼런스.md`
- `Coverage_및_검증_가이드.md`
- `Scoring_및_리포트_해설.md`

## 2. `문서/` 아래 분류 기준

### `문서/상태_및_이력`

현재 구현 자체보다 과거 상태, 업그레이드 이력, deprecated 안내가 중심인 문서를 둔다.

배치 기준:

- 예전 계획이나 전환 메모지만 삭제할 필요는 없는 문서
- 현재 source of truth는 아니지만 맥락 보존 가치가 있는 문서

현재 배치:

- `문서/상태_및_이력/codex4차.md`
- `문서/상태_및_이력/codex5.3_업그레이드_프롬프트.md`

### `문서/초기_설계`

현재 구현과 일치하는 운영 문서가 아니라, 초기에 검토했던 설계 구상이나 상위 평가 프레임 문서를 둔다.

배치 기준:

- 현재 CLI/스키마/카탈로그와 1:1 대응되지 않는 설계 문서
- 배경 설명이나 프레임 참고용으로만 남길 문서

현재 배치:

- `문서/초기_설계/Layer_1_2_3_4_설계.md`

### `문서/조사_및_분류`

조사 자료 통합본, taxonomy 초안, 유형 분류 자료처럼 구현 설명보다 분석/정리 목적이 큰 문서를 둔다.

배치 기준:

- 코드 동작 설명이 아니라 공격 유형 정리나 source survey가 중심인 문서
- 현재 샘플/카탈로그보다 넓은 개념 공간을 다루는 문서

현재 배치:

- `문서/조사_및_분류/최종프롬프트인젝션 유형.md`

## 3. 새 문서를 추가할 때 기준

- 현재 코드 동작을 설명하고 운영자가 바로 써야 하면 루트에 둔다.
- 과거 계획, 비교 메모, 업그레이드 기록이면 `문서/상태_및_이력`에 둔다.
- 구현 전제보다 상위 구조를 설명하는 설계안이면 `문서/초기_설계`에 둔다.
- 유형 조사, taxonomy 확장안, 외부 자료 통합본이면 `문서/조사_및_분류`에 둔다.

## 4. 판단이 애매할 때 우선순위

- source of truth에 가까운가
- 현재 구현과 직접 대응되는가
- 운영자가 첫 진입 시 바로 읽어야 하는가

이 셋이 모두 아니면 루트보다 `문서/` 하위 폴더에 두는 편이 맞다.
